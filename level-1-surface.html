Level 1 · Table stakes

The Surface

Does the user instantly understand what they see? This is everything visible on screen. Most failures here aren't code bugs. They're comprehension bugs. The user can see everything and still not understand it.

1

Have I solved the two-state problem?

For any control that switches between states, does it show the current state or the action that will happen? Am I consistent about this across the whole product? If there's any ambiguity, have I separated status (a clear indicator of where things are now) from action (a control that changes it)? A user should never have to guess what state the system is in.

2

Am I using established patterns instead of inventing new ones?

Users decode familiar patterns in a fraction of a second because they've seen them thousands of times. Every novel interaction carries a learning cost. If I've invented something new, do I have a very good reason, and does it survive a 5-second glance test with someone who's never seen it?

3

Does the presentation communicate the behavior?

Identical content reads completely differently depending on how it's grouped, framed, and shaped. Have I used layout and visual structure so the user knows what's clickable, what's switchable, and what's purely informational, without labels explaining it?
